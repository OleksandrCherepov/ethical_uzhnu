Структура звіту OSINT

Executive Summary

  • Scope — межі та цілі розвідки
  • Ключові знахідки — топ-5 критичних результатів
  • Оцінка ризику — загальний рівень загрози

Детальні знахідки

  • Категоризація — за серйозністю (Critical → Low)
  • Докази — скріншоти, URL, метадані
  • Пояснення ризику — як це може бути використано
  • Рекомендації — конкретні кроки для усунення

Запам'ятайте

Звіт — головний продукт розвідки. Без якісного звіту вся зібрана інформація залишається даремною. Замовник оцінює роботу за звітом, а не за кількістю використаних інструментів.

OPSEC — операційна безпека

Анонімність

  • VPN з no-log політикою
  • Tor Browser для веб-розвідки
  • Whonix / Tails OS
  • Окремі віртуальні машини
  • Мережевий трафік через проксі

Ідентичність

  • Alias / псевдоніми для операцій
  • Окремі акаунти для розвідки
  • Burner phones для верифікації
  • Тимчасові email-адреси
  • Ніколи не змішувати з особистим

Браузер

  • Disable WebRTC (витік IP)
  • Block third-party cookies
  • uBlock Origin — блокування трекерів
  • NoScript — контроль JavaScript
  • Регулярне очищення кешу

Типова помилка

Використання особистих акаунтів для розвідки — найпоширеніша OPSEC-помилка. Один вхід з робочого профілю може розкрити вашу особу та скомпрометувати всю операцію.

Підсумок

Головна ідея

Розвідка — фундамент кожного пентесту. Якість зібраної інформації визначає ефективність усіх наступних етапів тестування на проникнення.

  • OSINT-методологія: планування → збір → аналіз → звітування
  • Перевіряйте 5 категорій: домени, витоки, люди, технології, документи
  • Звіт — головний продукт розвідки, структуруйте за серйозністю
  • OPSEC = анонімність + окрема ідентичність + захищений браузер
  • Ніколи не використовуйте особисті акаунти для розвідки
  • Документуйте кожен крок — це захист і для вас, і для замовника
