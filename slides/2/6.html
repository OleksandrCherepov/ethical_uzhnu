grep — пошук за шаблоном

grep — найважливіший інструмент фільтрації тексту

Шукає рядки, що відповідають заданому шаблону (регулярному виразу), у файлах або потоці даних.

# Базовий пошук
grep "password" config.txt

# Без урахування регістру
grep -i "admin" file.txt

# Рекурсивний пошук
grep -r "API_KEY" /var/www/

# Інвертований (виключити рядки)
grep -v "comment" code.py

# Розширений regex (OR)
grep -E "root|admin" /etc/passwd

# Витягнути IP-адреси з файлу
grep -o -E "[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}" file

Практичне застосування

grep -r "password" /var/www/ — один з перших кроків після отримання доступу до веб-сервера для пошуку облікових даних у конфігураційних файлах.

awk та sed

awk — обробка полів

# Перше поле
awk '{print $1}' access.log

# Користувачі системи
awk -F: '{print $1}' /etc/passwd

# IP невдалих входів
awk '/FAILED/ {print $11}' auth.log

# Підрахунок частоти
| sort | uniq -c | sort -rn

sed — потоковий редактор

# Заміна всіх входжень
sed 's/old/new/g' file

# Заміна на місці
sed -i 's/old/new/g' file

# Рядки 10-20
sed -n '10,20p' log

# Видалити коментарі
sed '/^#/d' config

Комбінування в конвеєр

cat access.log | grep "404" | awk '{print $1}' | sort | uniq -c | sort -rn — топ IP-адрес із помилками 404.

Підсумок

Головна ідея

grep, awk, sed + пайпи = потужний конвеєр обробки тексту для аналізу логів, пошуку вразливостей та створення звітів.

  • Три потоки: stdin, stdout, stderr
  • Пайпи (|) — ланцюжки обробки
  • grep — пошук тексту за шаблоном
  • awk — витяг полів та обробка даних
  • sed — потоковий пошук та заміна
  • sort | uniq -c | sort -rn — підрахунок частоти
