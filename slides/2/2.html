Принцип "все є файлом" та ключові каталоги

Базовий принцип Linux

В Linux все є файлом — текстові файли, каталоги, пристрої (/dev/sda), процеси (/proc/PID), мережеві сокети. Це дозволяє працювати з будь-яким ресурсом через єдиний інтерфейс читання/запису.

Каталог Призначення Значення для пентесту
/ Кореневий каталог, початок ієрархії Точка відліку для всієї файлової системи
/etc Конфігураційні файли системи passwd, shadow — облікові записи та хеші
/home Домашні каталоги користувачів SSH-ключі (~/.ssh/), історія команд
/root Домашній каталог root Привілейовані конфігурації та ключі
/var/log Системні логи auth.log — спроби аутентифікації
/tmp Тимчасові файли Запис дозволено всім — ескалація привілеїв
/proc Віртуальна ФС, інформація про процеси та ядро Версія ядра, змінні оточення процесів
/dev Файли пристроїв /dev/tcp — мережеві з'єднання з Bash
/bin, /sbin Системні бінарні файли Пошук SUID-біт бінарників для ескалації

Критичні файли для безпеки

Файли аутентифікації

  • /etc/passwd — структура: user:x:UID:GID:info:home:shell
  • /etc/shadow — хеші паролів, доступ тільки root
  • /etc/sudoers — правила привілеїв sudo

Файли для моніторингу

  • /var/log/auth.log — спроби входу в систему
  • /proc/version — версія ядра Linux
  • /etc/crontab — заплановані задачі
  • ~/.bash_history — історія команд користувача

Важливо для пентестера

Це перші файли для перевірки після отримання доступу до системи. Вони дозволяють зрозуміти конфігурацію, знайти облікові записи та визначити вектори ескалації привілеїв.

Підсумок

FHS — Filesystem Hierarchy Standard

Стандартна ієрархія файлової системи, яка є єдиною для всіх Linux-дистрибутивів. Знання структури каталогів — обов'язкова навичка для пентестера.

  • "Все є файлом" — базовий принцип Linux
  • /etc — конфігурація та аутентифікація
  • /var/log — логи для розслідувань
  • /tmp та /proc — ключові для ескалації привілеїв
  • /etc/passwd і /etc/shadow — перша ціль після компрометації
